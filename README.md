# Weather Analytics — Open-Meteo

Final project for **Analytics Engineering @ IE School of Science and Technology**.

An end-to-end analytics pipeline that extracts weather and air-quality data from the
[Open-Meteo API](https://open-meteo.com/), transforms it with dbt on DuckDB, and surfaces
the results in a Streamlit dashboard.

> **Status:** scaffolding stage. See [`Project_Instructions/open_meteo_group_project/TEAM_PLAYBOOK.md`](Project_Instructions/open_meteo_group_project/TEAM_PLAYBOOK.md) for the team plan and role ownership.

---

## Quickstart (TL;DR)

```bash
uv sync
uv run dbt deps
bash scripts/run_pipeline.sh         # extract → load → dbt build
uv run streamlit run streamlit_app/app.py
```

## 1. How do I run the extraction?

```bash
uv run python scripts/extract_open_meteo.py
```

Default cities: Madrid, Barcelona, Valencia, Sevilla, Bilbao. Override with:

```bash
uv run python scripts/extract_open_meteo.py \
  --cities Madrid Paris Berlin \
  --past-days 60 \
  --forecast-days 7
```

Outputs land in `data/raw/open_meteo/` as four CSVs:
`raw_locations.csv`, `raw_weather_daily.csv`, `raw_forecast_daily.csv`, `raw_air_quality_hourly.csv`.

> TODO (Role 1): `scripts/load_to_duckdb.py` will register the CSVs as DuckDB tables in a `raw` schema.

## 3. How do I run dbt?

```bash
uv run dbt debug         # verify connection
uv run dbt deps          # install packages
uv run dbt build         # run + test all models
```

The dbt profile is `weather_analytics` (defined in `profiles.yml` at the project root).
Adapter is `dbt-duckdb`, database file is `weather.duckdb`.
